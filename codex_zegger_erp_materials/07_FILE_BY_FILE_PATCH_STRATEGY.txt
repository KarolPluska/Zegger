ZEGGER ERP - FILE BY FILE PATCH STRATEGY
========================================

CEL DOKUMENTU
-------------
Ten plik mówi dokładnie jak przejść od obecnej wersji legacy do nowej wersji Zegger ERP.
Nie jest to tylko plan architektury. To jest praktyczna strategia plik-po-pliku:
- co przeanalizować
- co zachować
- co przepisać
- co rozbić na nowe klasy
- co usunąć z kalkulatora
- co zostawić jako kompatybilność
- jakie nowe pliki/foldery utworzyć

ŹRÓDŁA, KTÓRE MUSISZ PRZEANALIZOWAĆ
-----------------------------------
1. Folder legacy pluginu:
- zq-offer-suite/

2. ZIP legacy pluginu:
- zq-offer-suite-v1.2.18.7.zip

3. Kalkulator / launcher host:
- Źródło - Kalkulator ogrodzeń ZEGGER v1.9.9.1.html

4. Folder materiałów dla Ciebie:
- codex_zegger_erp_materials/

ZAŁOŻENIE GŁÓWNE
----------------
Budujesz nowy plugin Zegger ERP.
Nie rób kosmetycznej łaty starego pluginu.
Masz wykorzystać legacy jako bazę funkcjonalną, wzorzec i źródło danych, ale finalnie dostarczyć nową, pełną wersję systemu ERP.

Równocześnie:
- nie wolno stracić obecnego modułu Panel ofertowy
- nie wolno zepsuć jego flow, PDF, historii ofert i logiki pracy
- możesz całkowicie przebudować architekturę, jeśli zachowasz działanie i UX modułu ofertowego

REKOMENDOWANY MODEL PRACY
-------------------------
1. NIE edytuj legacy folderu jako finalnego runtime pluginu.
2. Utwórz nowy plugin folder, np.:
- zegger-erp/
3. Do nowego pluginu przenieś i przebuduj logikę legacy.
4. Legacy potraktuj jako źródło migracji, referencji i komponentów do adaptacji.
5. Jeśli potrzebujesz adapterów zgodności, zrób je świadomie i cienko.
